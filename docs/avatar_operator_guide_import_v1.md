# Avatar Operator Guide — Static Selector Import v1

## Purpose

This document explains the `AVATAR_OPERATOR_GUIDE_IMPORT_v1.csv` file and how to use it in Notion.

The **Avatar Operator Guide — Static Selector** is an operator-facing reference surface. It exists because the existing Avatar Context ID database in Notion only exposes bare IDs (e.g., `BOSMAX_AVP_0001`) without enough context for an operator to confidently select the right avatar for a given product, scene, or workflow.

This guide adds operator-readable metadata — scene summaries, pose descriptions, wardrobe style, use case, and Do / Do Not guidance — so that operators can make informed selections without touching the backend registry.

---

## What This Guide Is NOT

- **Not** the backend source of truth for avatar resolution. The authoritative registry is `registries/avatar_context_rotation.yaml`.
- **Not** a prompt generator. No raw prompt fragments are included.
- **Not** an expansion of the avatar registry. This PR does not create new Avatar Context IDs or modify the rotation resolver.
- **Not** a replacement for the runtime resolver. The resolver continues to use `avatar_context_rotation.yaml` directly.

## CSV Columns

### Safe Fields (21 columns, all present in import file)

| Column | Description |
|--------|-------------|
| Avatar Context ID | Unique identifier — use this in all BOSMAX workflows |
| Display Name | Human-readable label (Persona \| Wardrobe \| Scene) |
| Persona Label | Operator-safe persona name (Rizal / Azman) |
| Gender | Male / Female |
| Age Range | Estimated age range for the persona |
| Product Fit | Product this avatar is approved for |
| Silo | STEALTH or DIRECT lane |
| Scene Label | Short scene name |
| Scene Summary | Plain-language scene environment description |
| Mannequin Label | Pose label from mannequin library |
| Pose Summary | Plain-language pose and movement description |
| Wardrobe Style | Operator-safe wardrobe description |
| Best For | Conservative operator label for workflow matching |
| Content Type | Video / Poster / Both |
| Use Case | Short operator-facing explanation of when to use this avatar |
| Do | Safe operator instruction |
| Do Not | Clear warning against misuse |
| Safe Usage Notes | Operator-safe notes from registry |
| Status | APPROVED / SEED_READY / LOCKED |
| Source Registry | Source YAML file name |
| Source Version | Registry generation timestamp |

### Forbidden Fields (not present in import file)

The following fields are **never exported** to this operator guide:

| Forbidden Field | Reason |
|----------------|--------|
| `prompt_fragment_source` | Raw prompt path — internal only |
| `internal_notes` | Backend derivation notes — internal only |
| `compatible_physics_classes` | Internal resolver decisioning |
| Raw prompt fragments | Cannot be exposed to operator surface |
| Backend file paths (`avatars/`) | Internal repo structure |
| YAML internal references (`yaml::`) | Internal resolver format |
| Context IDs (`CTX_`) | Internal scene resolver keys |
| Mannequin IDs (`MAN_`) | Internal mannequin resolver keys |

## Notion Import Path

1. Open Notion → BOSMAX COMMAND CENTRE — ACTIVE OPERATOR PAGE
2. Create a new database: **Avatar Operator Guide — Static Selector**
3. Import CSV: `outputs/notion_import/AVATAR_OPERATOR_GUIDE_IMPORT_v1.csv`
4. Set column types:
   - `Avatar Context ID` → Title
   - `Status` → Select
   - `Silo` → Select
   - `Content Type` → Select
   - `Product Fit` → Select
   - `Gender` → Select
   - All other text columns → Text
5. Create views as needed (filter by Silo, Product Fit, Status)

## Validation

Run before import:

```bash
python scripts/validate_avatar_operator_guide_import.py
```

The validator checks 14 points:
1. CSV exists
2. Exactly 10 rows
3. All 21 required columns present and in order
4. 8 BOSMAX rows
5. 2 MWCB rows
6. No forbidden columns
7. No forbidden text fragments anywhere in file
8. Every row has Scene Summary
9. Every row has Pose Summary
10. Every row has Safe Usage Notes
11. Every row has Status
12. Source Registry = `avatar_context_rotation.yaml`
13. No duplicate Avatar Context IDs
14. All 10 expected IDs present

Expected output: `✅ VALIDATION PASSED`

## Remaining Gap — Future Work

This PR covers **existing approved avatar context packs only** (10 rows).

The following expansions are **not in scope for this PR** and require a separate `Avatar Registry Expansion v2` task:

- Additional personas (NORA, SARA, or other female personas)
- Additional scene types (café, outdoor market, hospital corridor)
- Additional wardrobe variants (corporate formal, hijab variants)
- LIFESTYLE silo avatar packs
- SAVAGE_HPAS silo avatar packs
- YouTube Shorts / Shopee / Lazada platform-specific avatar packs

Any avatar count expansion must go through `bosmax-product-registration` and the avatar schema pipeline before being added to `avatar_context_rotation.yaml` and reflected in a future version of this import file.
