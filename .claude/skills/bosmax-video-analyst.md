### PHASE 2 — PRODUCT B RESOLUTION + COMPATIBILITY CHECKS

→ Call **bosmax-product-intelligence** untuk resolve Product B data.

```

  product_name_B:      [dari user]

  silo_required_B:     [derive dari category + compliance_class]
  recommended_avatars: [dari product_record_B jika exist]
```

#### STEP 2B — 3 COMPATIBILITY CHECKS

**CHECK 1 — SILO COMPATIBILITY:**
```
silo_A (dari video): [STEALTH / DIRECT]
silo_B (Product B):  [derive dari category + compliance_class]

MATCH → ✅ Proceed. Keep avatar archetype dari A.

MISMATCH:
  → ⚠️ SILO CONFLICT
  → "Video reference guna [silo_A] silo.
     Produk [B name] memerlukan [silo_B] silo.
     BOSMAX akan adapt avatar + pronoun + formula.
     Production style dan scene kekal sama."
  → Force silo_B
  → Jika silo_A = STEALTH, silo_B = DIRECT:
     Avatar swap → suggest DIRECT avatar (SARA/JULIA/BELLA/MAK_TOK)
  → Jika silo_A = DIRECT, silo_B = STEALTH:
     Avatar swap → suggest STEALTH avatar (RIZAL/AZMAN)
```

**CHECK 2 — COMPLIANCE COMPATIBILITY:**
```
compliance_B dari product_record_B:

JIKA STEALTH_METAPHOR_REQUIRED:
  → All generated dialog untuk B mesti guna stealth metaphors
  → Forbidden tokens list activated
  → Dialog skeleton akan di-reconstruct dengan compliance filter
  → Flag all overlay text untuk compliance scrub

JIKA STANDARD:
  → Normal processing
```

**CHECK 3 — SCENE/CONTEXT COMPATIBILITY:**
```
scene_A: [dari Phase 1 — e.g., CTX_KITCHEN_MODERN]
category_B: [dari Product B]

Compatibility check:
  Product B category vs scene dari A — compatible?

JIKA incompatible:
  → Suggest 3 alternative scenes
  → Keep lighting + mood dari A, swap environment
  → User pilih

JIKA compatible:
  → ✅ Inherit scene dari A
```

**CHECK 4 — FORMULA COMPATIBILITY (Video-specific):**
```
formula_A: [detected dari Phase 1]
silo_B:    [dari Product B]

FORBIDDEN combos:
  SAVAGE_HPAS + MAK_TOK → swap to HSO atau PAS
  SAVAGE_HPAS + DIRECT silo product → swap to PAS atau HSO

JIKA conflict:
  → Auto-swap formula
  → Inform user: "Formula [SAVAGE_HPAS] tidak sesuai untuk [product B category].
    BOSMAX auto-swap ke [HSO/PAS]. Boss boleh override jika mahu."
```

#### STEP 2C — GENERATE NEW CONTENT DNA UNTUK PRODUCT B

```

  new_hook:
    → Ikut hook_pattern dari A (same structure)
```

```
→ Present generated hooks kepada user untuk approval sebelum proceed
```

## FAIL-CLOSED RULES

**HARD BLOCK — ABORT:**
- ABORT jika video/frames tidak dapat dianalisa
- ABORT jika Product B tidak resolved
- ABORT jika scale_anchor_B = null + platform = TikTok
- JANGAN output any verbatim dialog dari video A ke dalam Product B script
- JANGAN carry forward brand claims dari A ke B
- JANGAN generate Block 2+ tanpa Block 1 end-state (multi-block rule)

**AUTO-HEAL:**
- Silo conflict → auto-adapt, inform user
- Formula conflict → auto-swap, inform user
- Scene incompatible → suggest alternatives
- Copywriting B kosong → generate original, seek approval
- Avatar conflict dengan formula → auto-swap formula, log
