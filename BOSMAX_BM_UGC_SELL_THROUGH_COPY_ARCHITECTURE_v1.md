# BOSMAX BM UGC SELL-THROUGH COPY ARCHITECTURE v1

Authority file for fixing weak BM commercial UGC dialogue.

Purpose:
- stop flat "nice recommendation" scripts that do not convert
- force Hook/Pain/Friction/Relief/CTA logic into commercial BM UGC video
- give GROK multi-block video a dialogue bridge so Block 2 does not feel late or detached

---

## 1. Core Diagnosis

Weak BM UGC video usually fails in two ways:
- dialogue sounds like soft commentary instead of sales copy
- Block 2 restarts with dead air or a fresh sentence, so the seam feels laggy and lipsync looks off

Therefore BOSMAX must lock:
- commercial copy arc
- dialogue bridge
- WPS-safe spoken density

---

## 2. Mandatory Formula Families

For BM commercial / recommendation / TikTok UGC video, choose one of these:

### A. SELL_THROUGH_HPFRC

Use when the goal is:
- direct recommendation
- faster conversion
- practical household / product demo

Structure:
- `H` = Hook
- `P` = Pain
- `F` = Friction
- `R` = Relief / product reason
- `C` = CTA

Plain meaning:
- scroll-stop line first
- name the annoying problem
- make the problem feel inconvenient or costly
- introduce the product as the easier fix
- close with a direct action

### B. STORY_HSARC

Use when the goal is:
- softer storytelling
- emotional relatability
- gentler trust build before CTA

Structure:
- `H` = Hook
- `S` = Setup
- `A` = Agitate
- `R` = Relief / product turn
- `C` = CTA

Plain meaning:
- start with a familiar moment
- place the viewer into a real mini-scene
- increase the discomfort or inconvenience
- show the product as the turning point
- close with a natural action

---

## 3. Hard Copy Law

For BM commercial / household / recommendation UGC:
- at least one real pain or friction line must exist
- product cannot appear as a random mention with no tension before it
- CTA cannot be the only sales element
- dialogue cannot be only "sedap / senang / best / klasik / premium" filler

Forbidden weak patterns:
- polite product description with no pain
- generic comfort talk with no consequence
- benefit list with no hook
- Block 2 opening with a new calm sentence after dead air

---

## 4. Multi-Block Dialogue Bridge Law

For GROK multi-block BM commercial UGC:
- Block 1 must end with an unresolved spoken bridge or forward-moving clause
- Block 2 must finish or develop the exact same semantic thread within the first `0.5s–1.0s`

Block 1 end should sound like:
- a sentence that is still carrying momentum
- a thought that naturally wants completion
- a benefit lead-in that is not fully closed yet

Good Block 1 closing bridge examples:
- "...sebab yang paling aku suka,"
- "...yang buat aku terus simpan ni,"
- "...sebab bila kau perlukan cepat,"

Good Block 2 opening continuation examples:
- "...dia memang terus ada dekat tangan."
- "...tak payah tercari-cari lagi."
- "...kau terus boleh guna tanpa buang masa."

Bad seam examples:
- Block 1 fully closes, then Block 2 starts a totally new calm sentence
- Block 2 spends 2-3 seconds repositioning product before speech
- Block 2 opens with greeting or re-introduction

---

## 5. Duration Architecture

### 6s single block

Target use:
- punchy CTA clip
- one-problem one-solution quick sell

Preferred arc:
- Hook + Pain + Relief + CTA

### 10s single block

Target use:
- standard direct-response UGC

Preferred arc:
- Hook + Pain + Friction + Relief + CTA

### 10s + 6s

Target use:
- 16s GROK

Block plan:
- Block 1 `10s`: Hook + Pain + Friction + bridge-out
- Block 2 `6s`: bridge-in + Relief + CTA

### 10s + 10s

Target use:
- 20s GROK

Block plan:
- Block 1 `10s`: Hook + Pain + Friction + bridge-out
- Block 2 `10s`: bridge-in + Relief + proof/detail + CTA

### 6s + 6s

Target use:
- 12s GROK

Block plan:
- Block 1 `6s`: Hook + Pain + bridge-out
- Block 2 `6s`: bridge-in + Relief + CTA

---

## 6. BM Spoken Style Rules

BM commercial UGC should sound:
- conversational
- compressed
- specific
- first-person believable

Preferred texture:
- practical spoken Malay
- short clauses
- one idea per breath unit

Avoid:
- overly polished formal BM
- awkward literal translation from English
- too many abstract adjectives
- slow filler phrases that waste word budget

---

## 7. Reason-To-Believe Rule

Relief cannot be empty.

After the product enters the script, include at least one concrete reason:
- size convenience
- grip convenience
- easy reach
- easy store
- cleaner routine
- faster action
- less mess
- easier repeat use

This reason should feel observable, not mythical.

---

## 8. Copy Audit Checklist

Before releasing BM commercial UGC dialogue, pass all:

```text
☐ formula family declared: SELL_THROUGH_HPFRC or STORY_HSARC
☐ hook exists in the first spoken line
☐ pain or friction exists before or at product introduction
☐ relief includes at least one concrete reason-to-believe
☐ CTA exists
☐ if multi-block: Block 1 ends with bridge-out
☐ if multi-block: Block 2 starts with bridge-in inside first 0.5s–1.0s
☐ dialogue does not feel like flat commentary
☐ dialogue fits WPS safe max for the language and duration
```
